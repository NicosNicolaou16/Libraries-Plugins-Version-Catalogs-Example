This project is an empty template designed to accompany an article on Medium about the modern approach to managing dependencies in Android—**Version Catalogs**.

In this repository, Android developers can find a curated collection of commonly used libraries and plugins, organized for easy migration and scalability.

> [!IMPORTANT]  
> Check out the full migration guide here:  
> 👉 **[Migration Gradle Dependencies to Version Catalogs — libs.versions.toms (Part 3) - Medium](https://medium.com/@nicosnicolaou/migration-gradle-dependencies-to-version-catalogs-libs-versions-toms-part-3-ac3469155109)** 👈

## ✨ Key Features

*   **[Version Catalogs](https://developer.android.com/build/migrate-to-catalogs):** Centralized dependency management using the `libs.versions.toml` file.
*   **Dependency Sharing:** Learn how to share dependency versions across multiple modules efficiently.
*   **Plugin Management:** Modern way to handle Gradle plugins within the catalog.
*   **Best Practices:** Follows official Android recommendations for dependency structure.
